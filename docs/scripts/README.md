# Docs autogen scripts

Utilities that regenerate documentation content under `docs/` for the Fern site.

## Core Python API reference (hand-maintained)

The **NeMo Evaluator Core Python API** reference lives as static MDX under:

```
docs/references/api/nemo-evaluator/
  api/index.mdx
  api/api-dataclasses.mdx
  adapters/adapters.mdx
  adapters/adapter-config.mdx
  adapters/interceptors.mdx
  adapters/types.mdx
  sandbox/index.mdx
```

Edit those files directly. Pages match the structure of the [published reference](https://docs.nvidia.com/nemo/evaluator/latest/) — prose intros plus inline class/method tables and anchors.

`references/api/nemo-evaluator/cli.mdx` is also hand-maintained.

### Optional Griffe drafts

After Python API changes, you can emit Griffe drafts for comparison (not wired into CI or `make docs-autogen`):

```bash
cd docs
uv run --group docs python scripts/generate_api_reference_mdx.py --draft-dir /tmp/api-ref-draft
```

Merge any updates manually into the hand-maintained pages under `references/api/nemo-evaluator/`.

Filtering and grouping rules live in `generate_api_reference_mdx.py` (`API_DATACLASSES_GROUPS`, `ADAPTERS_AUTOSUMMARY`, `PUBLIC_METHOD_NAMES`, etc.) if you regenerate drafts.

## Benchmark catalog

Benchmark harness pages are generated by:

```bash
make docs-autogen
```

That runs `packages/nemo-evaluator-launcher/scripts/autogen_task_yamls.py --docs-format fern` only.

### Prerequisites

From the repo root:

```bash
cd docs
uv sync --group docs
```

### Preview and check

```bash
make -C docs docs          # fern docs dev
make -C docs docs-check    # fern check + md check + script lint/tests
```

CI runs the same checks in [`.github/workflows/fern-docs-ci.yml`](../../.github/workflows/fern-docs-ci.yml) after `make docs-autogen`.
