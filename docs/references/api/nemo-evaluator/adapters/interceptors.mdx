<span id="interceptor_reference"></span>

Reference for interceptor implementation classes. For usage and configuration, start with the [Adapters overview](/references/api/nemo-evaluator/adapters/adapters) and the [Configuration](/references/api/nemo-evaluator/adapters/adapter-config) classes — interceptors are enabled through `InterceptorConfig` entries in `AdapterConfig`, not by instantiating the classes below directly.

Each class documents its `Params` fields and the request/response hooks it implements.

<span id="cachinginterceptor"></span>

### `CachingInterceptor`

Caching interceptor is special in the sense that it intercepts both requests and responses.

| Field | Type | Description |
| --- | --- | --- |
| `headers_cache` | `Cache` |  |
| `requests_cache` | `Cache` |  |
| `responses_cache` | `Cache` |  |

<span id="cachinginterceptor-intercept_request"></span>

#### intercept_request

```python
intercept_request(self, req: AdapterRequest, context: AdapterGlobalContext)
```

Shall return request if no cache hit, and response if it is.

**Args**

- req (AdapterRequest): The adapter request to intercept
- context (AdapterGlobalContext): Global context containing server-level configuration


<span id="cachinginterceptor-intercept_response"></span>

#### intercept_response

```python
intercept_response(self, resp: AdapterResponse, context: AdapterGlobalContext)
```

Cache the response if caching is enabled and response is successful.



<span id="endpointinterceptor"></span>

### `EndpointInterceptor`

Required interceptor that handles the actual API communication. This interceptor must be present in every configuration as it performs the final request to the target API endpoint.
> **Important:** This interceptor should always be placed after the last request interceptor and before the first response interceptor.

<span id="endpointinterceptor-intercept_request"></span>

#### intercept_request

```python
intercept_request(self, ar: AdapterRequest, context: AdapterGlobalContext)
```

Make the actual request to the upstream API.

**Args**

- ar: The adapter request
- context: Global context containing server-level configuration


**Returns**

- AdapterResponse with the response from the upstream API

<span id="responsestatsinterceptor"></span>

### `ResponseStatsInterceptor`

Collects aggregated statistics from API responses for metrics collection.

Tracks the following statistics:
- Token usage (prompt, completion, total) with averages and maximums
- Response status codes and counts
- Finish reasons and stop reasons
- Tool calls and function calls counts
- Response latency (average and maximum)
- Total response count
- Number of runs, inference times (approximated by processing time from the first to the last response)

<span id="responsestatsinterceptor-intercept_response"></span>

#### intercept_response

```python
intercept_response(self, resp: AdapterResponse, context: AdapterGlobalContext)
```

Collect aggregated statistics from the response.

#### post_eval_hook

